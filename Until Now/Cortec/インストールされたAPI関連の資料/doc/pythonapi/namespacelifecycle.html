Lifecycle

In general there are two ways how object destruction is handled by the API.

  1. Manually, using the OBJECT_destroy function
  2. Automatically by another object handle

object destruction

Object handles have to be manually destroyed unless they are added to another object handle. The automatic destruction follows a hierachical pattern:

  1. Implant
  2. Stimulation command
  3. Stimulation function
  4. Stimulation atom

Each handle type in this list will automatically destroy all appended items from a lower level. Note that after adding a handle to a higher-level handle, deleting it manually will lead to an error! Object handles not contained will always have to be destroyed manually (except factory handles!).

listener

A special case is the implant listener. Though it is registered on an implant handle, it will not be automatically destroyed by that implant. Therefore, it has to be deleted manually.

When destroying a listener handle, it must not be registered on an implant. If it should be deleted you first have to:

  1. unregister it from the implant handle
  2. destroy the implant handle (which will unregister the listener)

handles

Factory handles are managed by the API itself and therefore there is no way to destroy them manually
