1 '''@doctring stimulationcommand
2 
3  A stimulation command defines a sequence of stimulation functions,
4  which are executed one after the other by the implant.
5 
6  This is a generic interface that is intended to be used with
7  different types of implants with different stimulation capabilities
8  each. Stimulation capabilities may vary regarding the number of
9  channels used for stimulation, stimulation amplitude, form of the
10  stimulation signal over time. To reflect this, a stimulation
11  command consists of a sequence of stimulation functions. A
12  stimulation function can be composed arbitrarily complex.
13 
14  Each command holds a tracing id that can be set arbitrarily. This id is used to identify executions of the
15  command in the application logs.
16 
17  The execution of a command can be executed internally by setting a number of repetitions > 1. This number of
18  repetitions differs from the repetitions of stimulation functions, since all functions in the command are repeated.
19 
20  Example: Given a command that contains stimulation function A and B and a repetition number of 3. Then, the
21  command functions will be executed as follows:
22 
23  A | B | A | B | A | B
24 
25  If for the function A, the repetition number is set to 2 additionally, the execution changes to:
26 
27  A | A | B | A | A | B | A | A | B
28 
29  Typical usage of a stimulation command:
30  1. Create an empty stimulation command instance (with stimulation
31  command factory)
32  2. Repeatedly add stimulationFunction instances
33  3. Send stimulation command to implant by calling
34  implant_startStimulation().
35 '''
